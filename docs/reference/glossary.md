# Glossary

Common terms used in UCL's enterprise architecture documentation.

API
: Application Programming Interface. A defined way for systems to communicate.

Business Capability
: A description of what an organisation does, independent of how it is structured or what technology it uses.

COTS
: Commercial Off-The-Shelf. Software products available for purchase rather than custom-built.

DPIA
: Data Protection Impact Assessment. A process to identify and minimise data protection risks.

Enterprise Architecture
: A discipline for proactively and holistically leading enterprise responses to disruptive forces by identifying and analysing the execution of change toward desired business vision and outcomes.

ITSM
: IT Service Management. The practice of managing IT services to meet business needs.

JANET
: Joint Academic Network. The UK's national research and education network.

RBAC
: Role-Based Access Control. An approach to restricting system access based on user roles.

RTO
: Recovery Time Objective. The maximum acceptable time to restore a service after disruption.

RPO
: Recovery Point Objective. The maximum acceptable amount of data loss measured in time.

SaaS
: Software as a Service. Software delivered over the internet on a subscription basis.

SITS
: Strategic Information Technology Services. UCL's student records system.

SSO
: Single Sign-On. An authentication scheme that allows users to log in once and access multiple systems.

VLE
: Virtual Learning Environment. A platform for delivering online teaching (e.g., Moodle).
